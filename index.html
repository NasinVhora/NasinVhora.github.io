KRUG’S FIRST LAW OF USABILITY

Usability means users can easily access and understand the controls of webpages without any confusion. For e.g: For Submit or Save button, we can use green color. For Cancel button use "red" color.

Scanning, Satisficing, And Muddling Through

Scanning = Users are not intereseted to read all the contents of the webpage.Focus on the main contents rather than other things. Satisficing, And Muddling = Satisfy the user by fullfill the requirement of webpage.

Designing For Scanning, Not Reading

Make webpages effectively and efficiently. User wants to read important informations only. Focus on the main point.

Why users like mindless choices

Create that webpage which has a specific meaning of dropdown and button. User doesn't like to read all the contents or click many times. Be specific for your actions.

The art of not writing for the web

Write easy and informative contents of the webpage, so that users can easily understand what we try to explain. Your contents must have useful information.

Street signs and Breadcrumbs: Designing navigation

Use navigation in your webpage. By using navigation, we can easily move forward or backword to the webpages. Use "Link" if you directly want to move the another page.

The Importance Of Getting People Off On The Right Foot

Home page is one of the import thing in a website. Our home page should be designed in such a way that every item is placed in orderly manner which will be easy to check for a visitor. Two things which will explains about our website. One is tagline which will be placed under the logo of the web site. It describes about our web site. The welcome blurb which is displayed in the promotional section.

The Farmer and the Cowman Should Be Friends

Every person in the designing team feel that usability testing is waste of time and they try to spend most of the time on one issue over and over. They feel that what every they like on a web site will be liked by everyone which is a false myth. It’s better to hire set of people to do usability testing.

Keeping Testing Simple

The author says usability testing once a month is sufficient. It is not necessary that user must belong to the target group. If novice user is facing a problem the actual user will surely face the same problem. On the day of the test, help the user to understand the task and conduct test. Ask questions regarding the test and pay them.

Mobile

Mobile websites are not so different from the desktop websites. The only difference is that the users scan the mobile sites much faster. The mobile device has less space compared to the desktop, so adjusting every elements which were visible on the desktop site is not possible most of the times. Developers these days generally use mobile first approach, i.e. element extra information for the mobile site. Mobile sites can be as deep as the desktop sites, it all depends on the developer. Some things to keep in mind is that there are no cursors in mobile so effects on hover for tool-tip and such cant be visible in the mobile. A developer has to carefully replace those functions.

You don’t see any elephants around here, do you?: What Do-It-Yourself Usability Testing is, Why It Always Works, and Why So Little of It Gets Done

Usability testing is watching people use the product you have built in order to: 1. Make it easier for them to use 2. Proving it is easier to use 3. This is different from a quantitative test, where you try to measure things like success rate and checking how much time people finish a task you give them to do. 4. Do-it-yourself usability tests are for sure qualitative. Here the purpose isn't just to prove anything but to get insights on how to improve your site.

Go to www.rocketsurgerymadeeasy.com and watch the “Demo Test” file. At the end of the video, author asks us to list three usability problems which we think needs immediate fixing.

A morning a month, that’s all we ask: A Plan You Can Actually Follow

Here the author has devised a plan for us to use which will actually give good results. Author suggests to test every month and do testing for half a day. Author prefers to use a morning a month test then a big honkin test as big honking test requires a huge budget and preparation. Big Honking demands preparation of week and participants from the target audience. While morning test suggested by author takes half day to complete the test.

What do you test, and when do you test it?: Why the Hardest Part is Starting Early Enough

If the project is to redesign an existing site one should first test the current version of it. Author suggests to test site of competition to find out mistake in them, any other site having the same feature or functionality one is thinking to implement. A initial rough diagram of website’s design can be shown to participants and their viewpoint can be asked. A wireframe can also be shown to participant with rough sketch to know the viewpoint. After wireframes the participants can be asked to share their point of view.

Recruit loosely and grade on a curve: Who to Test with and How to Find them

For the initial test people not belonging to target audience can be recruited. Three participants each test are enough because they will face same problems. A screening can be conducted if a large pool of participants are available. While screening participants are informed what to expect from usability test. They should also be informed how they will be compensated for their time. Substitute participants must be recruited as they can be required at the end.

Find some things for them to do: Picking Tasks to Test and Writing Scenarios for them

List the task which must be tested. Prioritize the tasks. Create a brief description of each task termed as scenarios. Scenarios make it easy for the participants to understand the task. Participants should be not be allowed to use a search engine or any other site. A test of one scenario must be conducted to find out if there is any issue with the test setup. Scenarios should be printed for participants and facilitator.

Some boring checklists: And Why You Should Use them Even if, Like Me, You Don’t Really Like Checklists

Make arrangements for test like booking the room for test and observation, advertising for participants etc. Two weeks before the test date ask the stakeholders to check the task list and arrange goodies for participants. Screen participants and inform them about the test date and time. One week before test inform participants about the test room location and directions. Install screen sharing softwares. One day before test take confirmation from participant if they are attaining the test. Conduct a pilot test. On the day of test test the installed softwares before the test and conduct the test.

Mind reading made easy: Conducting the Test Session

Facilitator is one who conducts the test and observers the participant while testing. The test room must have screen recording and sharing software, a system to test the website, a microphone to record participant’s voice and a speaker phone. A facilitator must communicate with participants in short span of times and must be neutral. A facilitator must make sure that the participant is not annoyed and frustrated by them.

Make it a spectator sport: Getting Everyone to Watch and Telling them What to Look For

Convince the stakeholders and other people related to project to attain the usability testing. Prepare observation room by being sure there is computer with internet access, projector which displays screen of participant, speaker and speakerphone. Ask the observers to observe the test and make notes. Each observer should write three serious problems seen by them, ask facilitator to ask questions to the participant if they have any.

Comparing Notes and Deciding What to Fix

Start debriefing session by asking everyone to list three problems they though are serious. List all the problems mentioned by members in debriefing session. List the top ten serious problems which must be corrected. Sort them in descending order of their worstness. Bring the members back on track if discussion is diverted on some other topic.

Why Doing Less is Often the Best Way to Fix Things

Problems should be fixed by making adjustments on the design rather than starting from the scratch. Some elements can be removed from the websites to improve the usability instead of adding something on the website to solve a problem.

The usual suspects: Some Problems You’re Likely to Find and How to Think about Fixing them

A user can be distracted if there are too many elements in home page and be confused and can set to wrong directions to find the information. To make sure user navigates to right place home page should be tested in every session.
